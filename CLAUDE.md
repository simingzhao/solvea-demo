# CLAUDE.md — Solvea Demo (AI Receptionist Builder)

## Project Overview
A demo inspired by Solvea.cx — an AI Receptionist platform where users create, test, and deploy AI customer service agents. The core differentiator: we use **Claude Agent SDK** for the Builder and **Anthropic Messages API** for the runtime.

**Purpose**: Demo for a meeting with the Solvea founder. Show that one person can build the core product.

## Tech Stack
- **Framework**: Next.js 15 (App Router)
- **Styling**: Tailwind CSS + shadcn/ui
- **Builder Backend**: Claude Agent SDK (`@anthropic-ai/claude-agent-sdk`) — generates agent configs
- **Runtime Backend**: Anthropic SDK (`@anthropic-ai/sdk`) — runs agent conversations
- **Knowledge Base**: In-memory vector store (OpenAI Embeddings or simple TF-IDF)
- **State Management**: Zustand
- **Database**: None (localStorage + in-memory for demo)
- **Deploy**: Vercel

## Architecture

### Two-Layer Design

**Layer 1: Builder (Agent SDK)**
- User describes their business in natural language
- Agent SDK auto-researches (WebSearch/WebFetch), asks follow-up questions
- Outputs a structured AgentConfig JSON
- User's own API key is used

**Layer 2: Runtime (Messages API)**
- Takes the AgentConfig and runs customer conversations
- Tool calling for: knowledge search, appointment booking (mock), transfer to human
- SSE streaming responses
- User's own API key is used

### API Key Flow
- User enters Anthropic API key on first visit
- Stored in localStorage (never persisted server-side)
- Sent via request headers to API routes
- API routes use it to init SDK/client per-request

## Key Implementation Notes

### Builder System Prompt
The Builder is itself an Agent. Its system prompt instructs it to:
1. Greet the user and ask about their business
2. If user provides a URL, use WebSearch/WebFetch to research
3. Ask 3-5 targeted follow-up questions
4. Generate a complete AgentConfig JSON wrapped in `<agent_config>` tags
5. Present a summary and ask for confirmation
6. Allow iterative refinement through continued conversation

### Agent SDK Usage (Builder)
```typescript
import { query } from "@anthropic-ai/claude-agent-sdk";

// The SDK reads ANTHROPIC_API_KEY from env
// For per-request user keys, we set process.env temporarily
// or use child_process with custom env
for await (const message of query({
  prompt: userMessage,
  options: {
    systemPrompt: BUILDER_SYSTEM_PROMPT,
    allowedTools: ["WebSearch", "WebFetch"],
    permissionMode: "bypassPermissions",
  }
})) {
  // stream to client via SSE
}
```

### Runtime (Playground/Widget Chat)
```typescript
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic({ apiKey: userApiKey });
const stream = client.messages.stream({
  model: "claude-sonnet-4-20250514",
  system: agentConfig.systemPrompt,
  messages: conversationHistory,
  tools: agentConfig.tools,
});
```

### Tool Execution (Mock for Demo)

- Standalone React component, bundled separately
- Injected via `<script src="/widget.js" data-agent-id="xxx">`
- Floating bubble → expandable chat panel
- Communicates with same /api/chat endpoint
